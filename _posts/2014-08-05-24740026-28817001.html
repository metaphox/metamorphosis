--- title: 如何评价新出的编程字族 Input？ layout: post --- 十年前比较懵懂，最喜欢用 Notepad++ 配上没有 ClearType 的 Georgia 写程序，非等宽字体和 old style number 搭配起来加上语法高亮效果其实挺好。我来盗一张 StackOverflow: Are there any programming fonts with oldstyle figures? 问题答案中配的图：


图中小字号部分在 Windows 下面关掉 ClearType 其实看起来没这么不堪，没记错的话 12 号到 14 号都应该算是锐利清爽。0 和小写 o 不容易分辨是个问题，不过印象里没有因此有过太多困扰。

后来转用 Linux，因为终端模拟器多半（全部？）不支持 proportional，折腾一番之后认命只用 Courier New 作为编程字体，从此再也没动过重启 proportional 的念头。切换到 Mac 后觉得 Monaco 挺好，之前在 @Rio 蛊惑下用过一阵子 Source Code Pro，个人觉得实在是太宽，所以目前用的是 Pragmata，这款字体除了美观之外真是够瘦，并排显示两个 80 列宽的终端毫无问题。

昨天下载了 Input 试用，由于选择太多，一时还没有找到一个舒服的，不过至少觉得 Mono Compress Regular 这一款不输 Pragmata：
有一个不太舒服的地方是它的花括号（ { ）和方括号（ [ ）比较难以区分。

至于 proportional 部分，试着用了一番 Input Sans / Serif，乍看之下的感觉是「这真的是 proportional？」，字符宽度差异还是太小，像 Gerogia 那样的非等宽美感（恶趣味？）并没有凸显出来。以这种近似等宽的观感，对换不能对齐、块选择参差的缺憾，我可能一时还接受不了。
